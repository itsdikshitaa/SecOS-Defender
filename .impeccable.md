## Design Context

### Users
SecOS Defender is built for security analysts, blue-team engineers, and incident responders who need a self-hosted endpoint defense console for triage, investigation, and controlled response across Linux and Windows hosts. They use it in high-attention environments where signal density matters more than decoration.

### Brand Personality
Decisive, forensic, disciplined. The interface should feel trustworthy, high-signal, and operationally sharp rather than flashy or playful. The emotional target is confidence under pressure.

### Aesthetic Direction
Use a bright industrial-editorial visual system instead of the default dark SOC trope. Favor warm paper-tinted surfaces, steel and ink neutrals, red/orange risk accents, strong typography, and asymmetric control-room layouts. Avoid neon-glow dashboards, generic card grids, and interchangeable SaaS styling.

### Design Principles
1. Lead with signal: the most important risks and changes should dominate the page immediately.
2. Keep operational rhythm: dense data is acceptable, but grouping and spacing must make scanning effortless.
3. Make severity tactile: state changes, emphasis, and motion should reinforce urgency without visual noise.
4. Prefer investigation over ornament: every graphic and interaction should improve analyst comprehension.
5. Stay credible: the interface should feel like a serious security product, not a demo skin.
